InfoSynth: Information-Guided Benchmark Synthesis for LLMs

Ishir Garg, Neel Kolhe, Xuandong Zhao*, Dawn Song
University of California, Berkeley
*Corresponding author
Code Dataset Thread arXiv

InfoSynth leverages information theory to synthesize novel, diverse coding benchmarks for LLMs.

Abstract

Large language models (LLMs) excel at reasoning and code generation, yet creating contamination-free, diverse benchmarks to evaluate these capabilities remains difficult and costly. InfoSynth introduces an information-theoretic framework and scalable pipeline for automatically synthesizing robust Python coding benchmarks. We quantify novelty via KL divergence and diversity via differential entropy on text embeddings, avoiding expensive model evaluations. Building on these metrics, we generate problems via genetic mutation and crossover, self-verify solutions and tests in a code execution environment with iterative feedback, and deduplicate using MinHash+LSH. InfoSynth produces benchmarks with higher novelty/diversity than seeds, accurate tests/solutions for ~97% of problems, and enables control over novelty, diversity, and difficulty.

Case Study

InfoSynth mutations adjust problem difficulty and crossovers compose novel constraints:

  • Mutation (hard): Sum an array where entries may include nested lists at any depth.
  • Crossover: Rotate a list by k while ensuring the sum of any 3 consecutive elements in the rotated list does not exceed a threshold.

Each generated problem includes auto-produced tests and a verified solution via iterative code feedback.

Overview

InfoSynth provides a self-verifying benchmark synthesis pipeline for LLMs that closely mirrors the structure of AgentSynth pages while preserving this site's styling. Our contributions:

  • Define novelty (KL divergence) and diversity (entropy) metrics over text embeddings.
  • Genetic mutation (easy/medium/hard) and crossover to generate new problems.
  • Iterative code feedback: auto-generate solutions and tests, execute, refine until all tests pass.
  • K-farthest filtering to boost novelty/diversity; MinHash+LSH to deduplicate.
  • Post-processing to resolve edge cases and improve clarity without changing problem intent.

Data Composition

InfoSynth produces multiple datasets seeded from MBPP and LeetCode. Typical statistics:

  • Generation size: ~990–1007 problems per run; filtered: ~170–572 by novelty/diversity and deduplication.
  • Tests per problem: ~8–10 asserts; coverage: ~99–100% when all tests run.
  • Human verification: ~96–98% correctness on sampled problems after post-processing.

Experiment Results

  • Accuracy: Generated problems include tests/solutions that are correct ~97% of the time with ~99–100% coverage.
  • Quality: Datasets consistently show higher novelty and diversity than seeds; filtering/post-processing further improve diversity.
  • Control: Mutation difficulties modulate dataset difficulty; k-farthest selection increases novelty/diversity.
  • Efficiency: Metrics avoid costly test-taker evaluations while aligning with human intuition across datasets.

Task Generation Process

Starting from seed datasets (e.g., MBPP, LeetCode), InfoSynth generates multiple variants per seed via mutation and crossover, selects low-similarity candidates (k-farthest), and verifies each with auto-produced solutions and tests in a Python execution environment. Failures trigger iterative feedback where the model revises code/tests using full error history. Passing problems are deduplicated (MinHash+LSH) and lightly post-processed for edge-case clarity.

Novelty & Diversity Metrics

We embed problems with sentence transformers and project with UMAP for reliable distance estimates. Novelty is estimated via a k-NN KL-divergence estimator comparing new vs. seed distributions; diversity uses the Kozachenko–Leonenko differential entropy estimator with larger k to capture global structure. This yields efficient dataset quality assessments without model evaluations and supports optimization of novelty/diversity.
